KG Views That Feel Like Finished Work

These concepts are not generic graph demos. Each one is designed around the actual comparison task in this project: gold-standard clinical knowledge graph versus the graph behaviour extracted from LLMs.

How to choose

Use the atlas if the audience is primarily clinicians and you want the graph to read like decision logic.
Use the editorial panel if the audience is reviewers, readers, or slides and you need a high-status figure.
Use the signal matrix if the priority is validation, parser auditing, or edge-level model comparison.
Recommended

Superimposed Graph

The graph view you actually asked for: KG1 stays visible and KG2 is drawn directly over the same nodes and connections.

  • Grey blueprint = not replicated.
  • Green overlay = recovered correctly.
  • Red overlay = wrong or spurious edge.

Clinician-first

Causal Atlas

A layered, domain-organized map that reads like treatment logic instead of a computer science graph.

  • Groups nodes by clinical domain and treatment family.
  • Edge thickness carries JSD strength; color carries correctness.
  • Works well in board review, app onboarding, and teaching.

Gallery Panel

A fixed-layout triptych designed for publication, slides, and “gold vs model vs model” comparison.

  • Same geometry in every panel, so differences become obvious.
  • Feels more like a journal figure than a software dashboard.

Analytic

Signal Matrix

A dense but elegant heatmap for edge validation, parser auditing, and model-to-model comparison.

  • Cell color = recovered status; tone = JSD magnitude.
  • Supports filters by family, treatment class, or edge type.
  • Strongest option for serious review work.
